# Messaging Guardrails: GMX Audit Control Center

## 1. Purpose
Use these rules for README, website copy, sales messaging, and release notes to keep claims accurate and evidence-backed.

## 2. Approved phrases
- deterministic and reproducible
- continuous security monitoring
- explainable triage
- submission-ready proof packaging
- real economic impact framing
- complements manual audits
- static-first, CI-compatible outputs

## 3. Avoid phrases
- guaranteed exploit detection
- fully automated security
- replaces manual smart contract audit
- unhackable or bulletproof
- complete coverage of all vulnerabilities
- real-time protection (unless implemented)

## 4. Evidence rules
- Every product claim should map to shipped behavior.
- Future features must be labeled Planned or Roadmap.
- Impact numbers must include context and methodology.
- Security score must include interpretation limits.
- Managed-service claims must match current operational capability.

## 5. Disclosure language
Include one advisory note in public docs:
"AI-assisted outputs are advisory and require human review. This suite does not guarantee exploit detection and is not a substitute for a full manual audit."

## 6. Severity and finding language
- Use candidate wording until human-reviewed validation is complete.
- Distinguish deterministic reproduction from exploitability judgment.
- Avoid implying guaranteed bounty payout outcomes.

## 7. Review checklist before publishing
- [ ] No forbidden claim language present.
- [ ] All feature claims link to shipped docs or outputs.
- [ ] Any roadmap item is clearly marked as not yet shipped.
- [ ] Advisory note is present in externally visible docs.
